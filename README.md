## 📝 Завдання до роботи
Розробити функціональність для статичних сторінок Web-додатку першої лабораторної роботи із використанням архітектурного шаблону MVC (Model-View-Controller). Обов'язково застосувати MVC для головної робочої сторінки додатка. В якості сховища даних використовувати `localStorage` браузера. Використання сторонніх Web-фреймворків (ReactJS, AngularJS, VueJS) заборонено.

**Обрана тематика:** Блог з коментарями (DevBlog).
Реалізувати можливість створення, перегляду та вилучення повідомлень блогу (назва, текстовий вміст), а також коментарів до кожного допису. Додатково реалізовано реєстрацію, авторизацію та динамічний профіль користувача.

## 🛠 Технологічний стек
* **Логіка та функціонал:** Vanilla JavaScript (ES6+)
* **Архітектура:** MVC (Model-View-Controller)
* **Збереження даних:** Web Storage API (`localStorage`)
* **Розмітка та стилізація:** HTML5, CSS3, Tailwind CSS

## 📂 Структура проекту
Проект розділено на логічні модулі для зручності підтримки коду:

**HTML-сторінки:**
* `index.html` — Головна робоча сторінка (Стрічка блогу)
* `profile.html` — Профіль користувача з можливістю редагування
* `about.html` — Інформація про додаток
* `login.html` — Вхід до системи
* `registration.html` — Реєстрація нового користувача

**JavaScript-модулі:**
* `app.js` — Точка входу, ініціалізація додатку
* `model.js` — Модель (робота з даними постів та коментарів у localStorage)
* `view.js` — Представлення (рендер DOM-елементів та прив'язка подій для блогу)
* `controller.js` — Контролер (зв'язок між Model та View)
* `auth.js` — Глобальна логіка реєстрації, авторизації та оновлення навігації
* `profile.js` — Логіка відображення та редагування даних поточного користувача
